# Порівняння роботи алгоритмів сортування в Python

## Теоретичний аналіз
| Алгоритм   сортування     | Найкращий випадок | Середній випадок | Найгірший випадок |
|---------------------------|-------------------|------------------|-------------------|
| Вставки                   | O(n)              | O(n^2)           | O(n^2)            |
| Злиття                    | O(n log n)        | O(n log n)       | O(n log n)        |
| Timsort                   | O(n)              | O(n log n)       | O(n log n)        |

## Експериментальне дослідження
Тестування виконано за допомогою модуля `timeit` на масивах розміром 1000, 5000 та 10000 елементів.  
Результати показали:
- Вставки ефективні лише на малих масивах.
- Злиття стабільно працює за O(n log n).
- Timsort перевершує інші завдяки комбінації двох підходів.

## Висновки
- Алгоритм вставок не підходить для великих масивів.
- Алгоритм злиття гарантує стабільну швидкість, але потребує додаткової пам’яті.
- Timsort є оптимальним вибором у Python, оскільки поєднує переваги обох методів і забезпечує найкращу продуктивність у більшості практичних випадків.
